# Changelog

## [Unreleased]

## [1.0.13] - 2026-07-23

### Added

- **Modular System Prompt & DeepSWE Evaluation Infrastructure**: Added `packages/deepswe-bench` evaluation harness with candidate system prompt arms (`arms/<name>.prompt.md`), automated binary compilation, read-only credential seeding, and Pier/Docker task container sandboxing with zero user/profile side effects.
- **Maintainer Evals Skill**: Created maintainer skill at `.veyyon/skills/evals/SKILL.md` and indexed in `.veyyon/skills/INDEX.md` for prompt tuning, feature overlays, and A/B benchmarking workflows.
- **Multi-Path Profile Instruction Discovery**: Updated profile `AGENTS.md` / `agent.md` discovery in `agents-guidance.ts` and `builtin.ts` to scan candidate paths in 4-tier descending priority order (`~/.veyyon/profiles/<profile>/agent/AGENTS.md`, `~/.veyyon/profiles/<profile>/AGENTS.md`, `~/.veyyon/profiles/<profile>/agent/agent.md`, `~/.veyyon/profiles/<profile>/agent.md`), with single-file `break` deduplication preventing per-profile duplicate prompt injection.
- **Subagent Frontmatter Tool-Filtering**: `executor.ts` intersects parent tool names with subagent frontmatter tool lists (`scout.md`, `reviewer.md`, `designer.md`, `librarian.md`), automatically stripping unused tool schemas (`edit`, `write`, `bash`, `launch`, `debug`) from `scout`'s system prompt and cutting subagent prompt footprint by 50%+.
- **Prompt Cache Stability Law & 3 CWD Mutation Vectors**: Documented the 3 working directory mutation vectors (`session.workdir` settings, `set_cwd` tool, `/cwd` slash command) and prompt cache protection rules across `docs/tools/set_cwd.md`, `docs/internal/retained-patterns.md`, and `docs/handbook/src/using/roles-and-profiles.md`.
- The auth broker is now configurable from `/settings`: the Global tab's new Auth Broker group holds **Auth Broker URL** and a write-only **Auth Broker Token** (a stored token renders as a mask and is never echoed; enter a new value to replace it, leave the mask to keep it, clear the field to delete it). Both persist nested under `auth.broker.*` in the machine-wide `~/.veyyon/config.yml`, and broker discovery now reads that global file as a per-key fallback after the profile's own config (env vars still win). Previously these keys were file/env-only.
- `Ctrl+B` (`app.bash.background`) moves the running foreground command to a background job. While a foreground `bash` call is waiting, the working line shows `‚ü¶ctrl+b background‚üß`; pressing it converts the run into a background job (`details.async.reason: "manual"`) whose result is still delivered automatically. When nothing is waiting the chord keeps its readline meaning (cursor left).

### Changed

- Fenced code blocks in the transcript no longer show their literal ``` markers. A block opens with a short rule and its language tag (`‚îÄ‚îÄ‚ï¥bash`) and closes with the bare rule, in the same dim tone the compaction and cache markers use. The raw backtick fences read as unrendered markdown even though the body was already syntax-highlighted.
- The footline's session-name chip now clamps to a fixed budget with an ellipsis. Auto-generated titles are sentence-length and an unclamped chip crowded out the model, mode, and context readouts sharing the row.
- Past prompts in the transcript now carry the composer's prompt glyph: a dim `‚Ä∫` gutter on the first line with a hanging indent on the rest. The prompt text renders in full silver on titanium (an earlier dim treatment read as gray-on-gray and made your own words the least readable text on screen), and while the agent is working a turn the active prompt's `‚Ä∫` turns ember so you can always see which message is being worked.
- The whole transcript now sits on one left rail shared with the composer: past-prompt glyphs at column 2, assistant prose, the Thinking label, and bash/eval headers, output, and status footers at the same inset. Bash/eval blocks lost their two full-width mode-colored rules; the mode color lives on the `$`/`>>>` header alone. Previously assistant text started at column 1, prompts at column 3, and tool chrome flush-left, so nothing lined up with anything.
- A cancelled turn's transcript line reads `Operation aborted` instead of the stacked `Error: Aborted: Cancelled`, and glob's truncation note no longer repeats the result cap twice (`limit 200 results, limit 200 results`).
- The footline's context bar is now a clean two-tone gauge: filled cells in the usage hue and dim outlines for the rest. The old resting bar mixed a shaded quarter-step glyph into the outlined track (it read as a stray rectangle, not data) and recolored arbitrary "major" cells gold; the percent text next to the bar already carries the precision. While the model is running the frontier cell pulses between filled and empty, doubling its cadence at the error level.
- The unicode symbol preset no longer uses glyphs with ambiguous or emoji-presentation widths (`‚ìò`, `‚è≥`, `‚èπ`, `‚è∏`, `‚í∏ ‚íø ‚ìÄ`). These render two cells wide in many fonts while the TUI counts one, so a status icon could swallow the space after it (`‚ìòwaiting on 1 job`). Replacements are narrow-safe: `i`, `‚ãØ`, `‚àé`, `‚Äñ`, and blank language tags matching their siblings.
- The composer no longer paints any background. The tinted "card" behind the prompt (in every form: the absolute hex, the OSC 11-derived tint, and the theme `composerBg` token) is removed; the input renders directly on the terminal's own background with just the hairline above and the metadata line below. The `composerBg` theme token is accepted but has no effect.
- The composer hairline and card outlines now derive their color from the terminal's reported background (OSC 11): a fixed contrast step above the real ground, so the chrome stays equally subtle on black, grey, and light terminals. The static near-black token remains the exact fallback when the terminal never reports its background.
- Skill, extension, and hook message cards now hug their content: the outlined frame shrinks to the widest line instead of always stretching to the terminal edge.
- Visible reasoning traces now open with a muted `Thinking` heading. With thinking display enabled, the trace rendered as bare italic prose with nothing separating it from the answer.
- The composer's idle placeholder now uses single spaces around its interpunct (`ask anything ¬∑ / for commands`); the old double spaces read as uneven gaps.
- The scratch-directory marker in the footline is now the quiet `‚ñ´` square and the MCP "connecting" mark is `‚ó¶` (the unfilled pair of the `‚óè` active mark). Both previously used `‚óå` (U+25CC DOTTED CIRCLE), which is the placeholder glyph fonts show under combining marks and read as a rendering artifact.
- The titanium theme no longer paints explicit black or near-black backgrounds anywhere in the terminal: user-message rows, tool output boxes, the status footline, and the composer card previously filled with `#000000`/`#0C0E12`, which assumed a pure-black terminal and rendered as harsh black slabs on any other terminal background. All of these now inherit the terminal's own ground. (Themes that do declare a `composerBg` still get the card, and its padding rows now carry the same ground as the input rows.)

### Fixed

- The repository is now self-contained: `argot` (the shorthand codec) is a workspace package at `packages/argot` instead of a `file:` dependency pointing five levels up into a monorepo path (`libs/context/argot`) that only exists on the author's machine. A fresh `git clone` and `bun setup`, and every CI job, could not run `bun install` at all because that path is absent, so nothing could be built or released from a clean checkout. The codec's public API is unchanged (consumers still `import ... from "argot"`).
- `veyyon update` and the startup auto-update now resolve the newest version from GitHub Releases (`api.github.com/repos/santhreal/veyyon/releases/latest`), the same source `install.sh` uses, instead of the npm registry. Veyyon has no npm package and never will, so the old check could never find a newer version and a binary installed from `get.veyyon.dev` could never self-update. The check fails loudly on a missing/rate-limited/malformed release instead of silently doing nothing.
- A source install (`install.sh --source`, whose `veyyon` on PATH is a symlink to the in-checkout launcher) is no longer at risk of being overwritten by a self-update: `veyyon update` prints how to update the checkout (`git pull && bun setup`) and exits cleanly, and the background updater skips it instead of failing on every launch. Previously the updater misread it as a plain binary and would swap the launcher for a downloaded binary.
- Fixed the composer card rendering as a cramped tinted strip hugging the input text. The rows of air above and below the input were bare spacers (terminal ground), so the card had no vertical body; they now carry the same card ground as the input rows. The input rows and padding rows also share one ground owner now, so the derived card can no longer stripe.
- Fixed the terminal-background detection never reaching the derived-tint chrome: the OSC 11 report now feeds the hairline, composer card, and card outlines, which previously always used their static fallbacks in the running app.
- Fixed the status footline re-serializing the streaming tool call's full arguments on every redraw: the context-usage memo's change detector `JSON.stringify`ed the arguments object per render tick (a streaming file write re-serialized its whole body dozens of times a second). It now uses an allocation-free structural size walk with the same change-detection behavior.
- Fixed the streaming glow ("the follow") painting the row's trailing padding instead of the text. Rendered rows are right-padded to the component width, and the trail selected the last cells of the padded row, so the accent glow landed on spaces and was invisible on every padded row. The trail now anchors at the last visible character and the padding passes through untouched.
- Fixed duplicated thinking/answer paragraphs sprayed into native terminal scrollback mid-stream. On the first delta after a streaming rebuild (e.g. the answer block appearing next to the thinking block), the smooth-reveal pacer restarted at zero for a Markdown child that already displayed hundreds of characters, momentarily collapsing its rows; the transcript's committed scrollback prefix diverged and, with `tui.scrollbackRebuild` off (the default), the re-anchor appended a second copy into history. The reveal now honors a monotonic contract: within one block, revealed text only ever grows (a takeover starts at the already-displayed length; a genuine rewrite still restarts from zero).
- Fixed a turn that errors before any streaming begins (the provider rejects the request at setup: unsupported thinking effort, bad model id, auth) dying silently in the TUI ‚Äî no working line, no banner, no clue. The pre-stream error now pins the same banner above the editor as a mid-stream failure.
- Fixed the `AgentSession` constructor storing the configured thinking level unclamped against the session's model. A persisted `high` landing on a reasoning model with no controllable effort surface (e.g. `devin/swe-1-6`) threw `requireSupportedEffort` at the first stream of every turn; the constructor now clamps via `resolveThinkingLevelForModel`, mirroring the session-restore path (`off` preserved, supported levels kept, dial-less models forward no effort).
- Changing the working directory mid-session with `/cwd` or the agent's `set_cwd` tool now re-roots the whole session, not just the filesystem cwd. Project settings, plugins, slash commands, capabilities, the ssh tool, and the system-prompt project framing reload for the new directory, matching `/move` (minus relocating the session file). Previously only tools moved and the rest of the session stayed pinned to the original project.

### Removed

- The `snap` compaction strategy (the experimental image-archive engine that rendered discarded history to bitmap frames) and its `@veyyon/snapcompact` package. `compaction.strategy` now offers two pure-LLM strategies, `summary` (the new default) and `handoff`. Sessions compacted by the old engine still open without loss: their archived plaintext source re-attaches to the compaction summary as recovered text, and the next compaction folds it into a normal LLM summary. A stored `snap` strategy value normalizes to `summary` on load.
