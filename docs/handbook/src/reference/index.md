# Reference

This part is the lookup desk: command flags, environment variables, exit codes, and the files Veyyon creates on your system. Use it when you need the exact name of a flag, the location of a config file, or the meaning of a return code.

- [CLI reference](./cli.md) — the `veyyon` command and every subcommand, plus common flags and config overrides.
- [Slash commands](./slash-commands.md) — every `/` command available inside an interactive session, grouped by task.
- [Tools reference](./tools.md) — model-facing tools (`sleep`, context, plugins, MCP resources, edit/exec) with params and safety.
- [Environment variables](./environment.md) — supported common variables grouped by purpose (location, auth, catalog, TLS, install, repair, terminal), plus descoped names.
- [Exit codes](./exit-codes.md) — the return codes Veyyon uses and how child-process exit status passes through.
- [File locations](./file-locations.md) — where Veyyon stores config, sessions, logs, and credentials under `~/.veyyon`.

Where to go next: if something is failing, see [Troubleshooting](../using/troubleshooting.md); for definitions of terms used throughout the book, see the [Glossary](../appendix/glossary.md).
